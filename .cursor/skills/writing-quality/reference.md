# Schreibqualitätsreferenz

Detaillierte Regeln für die `writing-quality`. Die Datei SKILL.md behandelt die gängigsten Regeln. Diese Datei enthält die vollständige Referenz für Zeichensetzung, Terminologie, Großbuchstaben und Prozedurformatierung.

&#x200B;---

## Interpunktion

### Zeiträume

- Verwenden Sie Punkte in einer Liste, wenn die Elemente einen Satz abschließen („Im Menü X können Sie:„)
- Punkte in einer Liste verwenden, wenn ein Element ein vollständiger Satz ist
- Punkte dürfen nicht mit dem Hauptinhalt formatiert sein (korrekt: Klicken Sie auf **Fertig**. / Falsch: Klicken Sie auf **Fertig.**)
- Verwenden von Punkten in Dateierweiterungen (.pdf)

### Kommas

- Oxford-Kommas immer: „Motiv, Mittel und Chance“
- Komma nach dem Einleitungssatz: „Klicken Sie auf „Öffnen“, um ein Dokument zu öffnen.“
- Komma nach dem Jahr in einem Datum Mitte Satz: „Der 2. Januar 2016, Ausgabe von…“
- Komma vor „then“ in den Prozeduren: „Klicken Sie auf **Setup** und dann auf **Schnittstelle**.“
- Kein Komma mit nur dem Monat und Jahr: „Januar 2016“

### Apostrophe

- Verwendung bei Kontraktionen: Nicht wahr, sie sind wahr, es ist wahr
- Singular Possessive: add&#39;s (Jens Computer)
- Plural Possessive endet in s: Nur hinzufügen &#39; (Der Computer der Jones)
- „it&#39;s“ (es ist) von „its“ (dazugehörig) unterscheiden

### Doppelpunkt, Kolon

- Kleinbuchstaben nach einem Doppelpunkt, es sei denn, gefolgt von einem direkten Zitat, Eigennamen oder Aufzählungsliste
- Verwenden am Ende eines Satzes/Fragments, der/das eine Liste einführt
- Kein Doppelpunkt nach den Verfahrensüberschriften
- Kein Doppelpunkt zur Einführung visueller Elemente oder Code-Segmente

### Semikolons

- Zwei Klauseln ohne Verbindung verbinden
- Getrennte Sätze mit eigenen Satzzeichen
- Kein fett gedrucktes Semikolon nach fett gedrucktem Text in Verfahren
- In einen Hyperlink nicht einschließen

### Anführungszeichen

- Sie werden als „Anführungszeichen“ (nicht als „Anführungszeichen„) bezeichnet
- Schließen von Anführungszeichen gehen außerhalb von Kommas und Punkten
- Die Platzierung eines Fragezeichens hängt davon ab, ob das Anführungszeichen oder der Satz die Frage ist

### Bindestriche

- Vermeiden Sie nach Möglichkeit Strukturen, für die Bindestriche erforderlich sind
- Trennzeichen zwischen zusammengesetzten Modifikatoren vor einem Substantiv: „Schreibgeschützter Speicher“
- Trennen, wenn ein Modifikator ein Partikel ist: „Kopiergeschützte Festplatte“
- Silbentrennung, wenn ein Modifikator eine Zahl ist: „12-Punkt-Schriftart“
- Keine Bindestriche in Tastenkombinationen - verwenden Sie + (Strg+I)

### em-Bindestriche

Verwenden Sie sparsam. Stattdessen sollte man den Satz neu formulieren.

### Bindestriche

- Bereiche: 2005-2013
- Minuszeichen oder negative Zahlen
- Verbinden eines zusammengesetzten Adjektivs mit einem anderen Wort: „Optionen vom Typ Dialogfeld“

### Ellipsen

- Ausgelassener Code oder Syntax
- Anzeige der Benutzeroberfläche, dass ein Dialogfeld angezeigt wird
- Ausgelassener Text in Anführungszeichen (fügen Sie einen Punkt hinzu, wenn Sie das Satzende ersetzen)

### Schrägstrich

- Kein Schrägstrich für Auswahlmöglichkeiten - verwenden Sie „oder“ („er oder sie“ oder nicht „er/sie„)
- Für Kombinationen: Ein-/Ausschalter
- Verwendung für Internetadressen und -teile

## Beispiele für Großbuchstaben

### Workfront-Begriffe - direkte vs. indirekte Referenz

| Begriff | Direkt (großschreiben) | Indirekt (Kleinbuchstaben) |
|------|---------------------|----------------------|
| Geplantes Abschlussdatum | „Füllen Sie das Feld **Geplantes Abschlussdatum** aus.“ | „Jede Aufgabe muss ein geplantes Abschlussdatum haben.“ |
| Layoutvorlage | „Navigieren Sie im Setup zu **Layoutvorlagen**.“ | „Weisen Sie jedem Benutzer eine Layout-Vorlage zu.“ |
| Gruppenadministrator | Suchen Sie das Feld **Gruppenadministrator**.“ | „Wenden Sie sich an Ihren Gruppenadministrator.“ |
| Benutzerdefiniertes Formular | „Fügen Sie das **Benutzerdefinierte Formular** an das Projekt an.“ | „Sie können für jedes Objekt ein benutzerdefiniertes Formular erstellen.“ |

### Allgemeine Workfront-Begriffe (werden beim Referenzieren der Benutzeroberfläche großgeschrieben)

- Benutzerdefiniertes Formular
- Dauer
- Geplante Stunden
- Geplantes Abschlussdatum
- Stundentyp
- Gruppenadministrator/Systemadministrator
- Workfront-Administrator (die Person, immer „Administrator“ in Kleinbuchstaben)
- Erinnerungsnachrichten
- Layoutvorlage
- Filter, Ansicht, Gruppierung (im Kontext von Filtern, Ansichten und Gruppierungen)
- Ressourcenplaner (Pläne)

### Schaltflächen

- UI-Groß-/Kleinschreibung folgen
- Wenn die Schaltfläche in der Benutzeroberfläche nur mit Großbuchstaben angezeigt wird, schreiben Sie sie stattdessen in Satzbuchstaben

## Terminologie

### Elemente der Benutzeroberfläche

| Element | richtiger Begriff |
|---------|-------------|
| Reduzierbare Liste von Optionen | Menü (bei Bedarf „Dropdown“ zur besseren Übersichtlichkeit hinzufügen) |
| Nicht verfügbare Befehle | „Nicht verfügbar“ (nie abgeblendet, „deaktiviert“ oder „grau„) |
| Popup mit Formularfeldern | Dialogfeld |
| Unbenannte Schaltflächen | Namen der QuickInfo der Schaltfläche verwenden |
| Fehlermeldungen/Popup-Benachrichtigungen | „Nachricht“ („Eine Meldung wird angezeigt, die Sie benachrichtigt…„) |
| Texteingabe | Feld |

### Rollen

| Kontext | Begriff |
|---------|------|
| Die Rolle in Workfront | Systemadmin |
| Die Person, die Workfront verwaltet | Workfront-Administrator |

### Überschreiben vs. Überschreiben

Das sind verschiedene Wörter mit unterschiedlichen Bedeutungen. Benutzen Sie das richtige für den Kontext.

### Dateierweiterungen

Verwenden Sie immer die Dateierweiterung, nicht das Akronym „Upload a .pdf file“ (nicht „Upload a PDF„).

## Prozedurformatierung

### Einführung

Nur bei Bedarf einbeziehen. Bevorzugte Formate (in der richtigen Reihenfolge):

1. Infinitivsatz + Doppelpunkt: „So erstellen Sie ein temporäres Kennwort für einen Benutzer in Ihrem System:“
2. Vollständiger Satz + Punkt: „Sie müssen temporäre Kennwörter für Benutzer in Ihrem System erstellen.“
3. Infinitivsatz + „Führen Sie diese Schritte aus“ + Doppelpunkt: „Um ein temporäres Kennwort für einen Benutzer zu erstellen, führen Sie die folgenden Schritte aus:“

### Schrittkonventionen

- Verwenden Sie nummerierte Listen für alle Verfahren, auch für einstufige
- Fett anklickbare Aktionen und Benutzeroberflächenelemente in Schritten
- Bestimmte Aktionsverben verwenden: „type“, „select“, „click“ — vermeiden Sie „set“ oder „specific“.
- Systemreaktionen vor oder unmittelbar nach ihrem Auftreten beschreiben

Beispiel für die Platzierung einer Systemantwort:

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### Bedingte und optionale Schritte

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### Unterschritte und Feldlisten

Beginnen Sie mit einem Befehl Imperativ (Konfigurieren, Angeben, Ausführen) und führen Sie dann die Listenfelder aus:

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### Multiple Methods

Dokumentieren Sie zunächst den einfachsten Weg:

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### Eckige Klammern

- Menü-/Registerkartennavigation anzeigen: „Klicken Sie auf **E** > **Benachrichtigungen**&quot;
- Fettgedruckte Schaltflächennamen, keine Klammern
- Leerzeichen um Klammern
- Nicht für Hauptmenü verwenden - verwendet das Hauptmenü-Snippet

## Fett und kursiv gedruckte Referenz

### Fett — Verwendet für:

| Kontext | In Schritten | Außenstufen |
|---------|----------|---------------|
| Anklickbare Schaltflächen/Links | Fett | Nicht gewagt |
| Menünamen | Fett | Nicht gewagt |
| Symbolnamen | Fett | Nicht gewagt |
| Namen von Dialogfeldern | Fett | Nicht gewagt |
| Seitennamen | Fett | Nicht gewagt |
| Feldnamen in Feldlisten | Fett + Doppelpunkt | Nicht gewagt |

### Kursiv — Verwenden Sie für:

- Text, den der Benutzer eingeben soll: &quot;*.workfront.com* in das URL-Feld eingeben“
- Nie für Hervorhebung - Stattdessen den Satz umstrukturieren

## Hinweise, Tipps, Warnungen — Detaillierte Anleitungen

### Verwendungsbereiche

**Hinweis** (`>[!NOTE]`):
- Informationen, die nicht zum Absatzkontext passen
- Vermeiden Sie Benutzerfrustration oder verschwendete Zeit („Sie benötigen eine Team-Lizenz, um diese Aufgabe zu erledigen.„)
- Versionierungs- oder Kompatibilitätsinformationen („Diese Funktion ist nur in der alten Proofing-Anzeige verfügbar.„)

**Tipp** (`>[!TIP]`):
- Vorschläge, die das Leben der Benutzenden vereinfachen, aber nicht entscheidend sind
- Beispiel: „Sie können mehrere Dokumente gleichzeitig freigeben, indem Sie die Umschalttaste gedrückt halten.“

**Wichtig** (`>[!IMPORTANT]`):
- Wichtige Informationen für den Schritt oder das Verfahren

**Warnung** (`>[!WARNING]`):
- Benutzer auf möglichen Datenverlust hinweisen

### Regeln

- Höchstens eine pro Abschnitt
- Stapeln Sie niemals mehrere Notizen/Tipps/Warnungen
- Verknüpfte Notizen zu einer mit Aufzählungszeichen kombinieren
- Keine Notizen verwenden, um neue Funktionen anzukündigen - den Artikeltext überarbeiten

## Querverweismuster

| Situation | Muster |
|-----------|---------|
| Nach Erteilung der Informationen | „Weitere Informationen finden Sie unter [Artikelname] (link).“ |
| Ohne vorherige Information | „Weitere Informationen finden Sie unter [Artikelname] (link).“ |
| So fügen Sie einen Abschnitt in einem anderen Artikel ein | „Weitere Informationen finden Sie unter [Abschnitt] in [Artikelname] (link).“ |
| So fügen Sie einen Abschnitt im selben Artikel ein | „Weitere Informationen finden Sie [Abschnitt] in diesem Artikel.“ |

Platzierung:
- In Absätzen: inline
- In Verfahrensschritten: Neue Zeile nach dem Schritt
- Mehrere Verweise: Aufzählungsliste

## DQTI-Qualitätsmerkmale

### Benutzerfreundlich

| Kennlinie | Schlüsselfragen |
|---------------|---------------|
| Aufgabenausrichtung | Konzentriert sich diese auf das, was der Benutzer tun kann? Ist die Überschrift ein zwingendes Verb? |
| Genauigkeit | Entspricht die Terminologie der Benutzeroberfläche genau? Sind die technischen Details korrekt? |
| Vollständigkeit | Gibt es genügend Informationen, um zu handeln? Gibt es unnötige Informationen, die entfernt werden müssen? |

### Leicht verständlich

| Kennlinie | Schlüsselfragen |
|---------------|---------------|
| Klarheit | Ist jeder Satz eine Idee? Sind Wörter einfach und direkt? |
| Konkretheit | Gibt es konkrete Beispiele? Benennen Sie Benutzeroberflächenelemente genau? |
| Stil | Aktive Stimme? Präsenz? Konsistente Terminologie? |

### Leicht zu finden

| Kennlinie | Schlüsselfragen |
|---------------|---------------|
| Organisation | Logische Struktur? Progressive Offenlegung? Neueste zuerst, wo zutreffend? |
| Abrufbarkeit | Kann ein Leser Überschriften scannen und den Inhalt verstehen? Werden Links bereitgestellt? |
| Visuelle Wirksamkeit | Listen für mehr als 3 Elemente? Tabellen für Vergleiche? Keine Textwand-Absätze? |

## Allgemeine Fehlerbehebungen

| Problem | Korrigieren |
|-------|-----|
| „Wir freuen uns, Ihnen Folgendes vorzustellen…“ | Marketing-Sprache entfernen - Funktion direkt angeben |
| „Die Konfiguration von X wurde aktualisiert“ | „Sie können jetzt X konfigurieren“ (aktiv, aufgabenorientiert) |
| „Es wird empfohlen, dass…“ | „Wir empfehlen…“ |
| „Benutzer können die Funktion verwenden, um…“ | „Sie können die Funktion verwenden, um…“ |
| „Wert festlegen“ / „Option angeben“ | „Wert eingeben“ / „Option auswählen“ |
| „Deaktiviert“ / „ausgegraut“ | „Nicht verfügbar“ |
| „Dropdown“ (eigenständig) | „Menü“ oder „Dropdown-Menü“ |
| „er/sie“ | „Sie“ oder „umstrukturieren“ |
| Gestapelte Notizen | In einer Notiz mit Aufzählungszeichen kombinieren |
| Fettdruck außerhalb der Verfahrensschritte | Fett entfernen |

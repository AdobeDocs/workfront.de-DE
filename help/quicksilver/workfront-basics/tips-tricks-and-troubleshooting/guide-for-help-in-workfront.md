## Wie können sich [!DNL Workfront]-Administratoren auf [!DNL Workfront] Produktversionen vorbereiten?

Diese drei Schritte können Ihnen als [!DNL Workfront]-Admin dabei helfen, jede neue Version im Auge zu behalten:

<table style="table-layout:auto"> 
 <col> 
 <col> 
 <tbody> 
  <tr> 
   <td><strong>Schritt 1: Erfahren Sie, was kommt</strong> </td> 
   <td> <p>Diese Ressourcen informieren Sie über Aktualisierungen, die für Workfront veröffentlicht werden:</p> 
    <ul> 
     <li> <p><a href="https://experienceleague.adobe.com/de/docs/workfront/using/product-announcements/product-releases/product-releases">Produktankündigungen</a>: Suchen Sie nach Updates der Produkt-Roadmap von Workfront? Auf der Seite mit Produktankündigungen finden Sie Versions-Updates und Roadmap-Präsentationen.</p> </li> 
     <li> <p><strong>Ankündigungszentrum</strong>: Überprüfen Sie Ihre Benachrichtigungen und erhalten Sie wichtige Updates von unserem [!DNL Workfront]-Produkt-Team zu Informationen für alle Kunden. Rufen Sie den Abschnitt <a href="https://experience.workfront.com/s/article/View-and-manage-in-app-notifications-323912892">[!UICONTROL „Notifications“]</a> in [!DNL Workfront] auf, und zwar in Ihrer oberen Navigationsleiste.</p> </li> 
    </ul> </td> 
  </tr> 
  <tr> 
   <td><strong>Schritt 2: Vorbereiten auf Aktualisierungen</strong> </td> 
   <td> 
    <ul> 
     <li> <p>Testen Sie die neuesten Aktualisierungen in Ihrer [!DNL Workfront] Sandbox-Vorschau-Umgebung. Informationen zur Sandbox-Vorschau-Umgebung finden Sie unter <a href="https://experience.workfront.com/s/article/The-Workfront-Preview-Sandbox-Environment-519456234">Die Sandbox-[!DNL Adobe Workfront]-Vorschau-Umgebung</a>.</p> </li> 
     <li> <p>Bestimmen Sie, wie sich diese Aktualisierungen auf die Arbeitsweise Ihrer Mitarbeiter auswirken. Unsere <a href="https://experienceleague.adobe.com/de/docs/workfront/using/home">Wissensdatenbank</a> und <a href="https://experienceleague.adobe.com/de/events"> Events</a> können helfen! Unsere Produkt-Manager nennen oft die direkten Endbenutzer oder geschäftlichen Probleme, die durch die Funktionalität gelöst werden, und helfen Ihnen, die Auswirkungen der Änderung abzukürzen.</p> </li> 
    </ul> </td> 
  </tr> 
  <tr> 
   <td><strong>Schritt 3: Kommunizieren Sie die Änderung</strong> </td> 
   <td> 
    <ul> 
     <li> <p>Senden Sie Benutzern in Ihrer Organisation eine Mitteilung darüber, was sich ändern wird, warum es für sie von Vorteil ist und mit wem sie sich bei Fragen in Verbindung setzen können.</p> </li> 
     <li> <p>Um die Arbeit zu verbreiten, sollten Sie die Identifizierung von Power-Benutzern oder Fachexperten in eigenen Bereichen des Tools aus einer Wissens- und Aktualisierungsperspektive erwägen. Dies hilft, die Interaktion zu fördern und die Akzeptanz zu unterstützen.</p> </li> 
    </ul> </td> 
  </tr> 
 </tbody> 
</table>

# Zugriff für Adobe Workfront Planning als eigenständiges Produkt erforderlich

>[!IMPORTANT]
>
>Die Informationen in diesem Artikel beziehen sich auf Adobe Workfront Planning, wenn es als eigenständiges Produkt gekauft wird. Lesen Sie diesen Artikel, wenn Ihr Unternehmen ein reines Adobe Workfront Planning-Paket erworben und kein Workfront Workflow-Paket gekauft hat.
>
>Informationen zu Adobe Workfront Planning beim Kauf zusammen mit einem Workfront Workflow-Paket finden Sie unter [Erste Schritte mit Adobe Workfront Planning](/help/quicksilver/planning/general/planning-overview.md).

In diesem Artikel werden Lizenzen, Zugriffsebenen und Benutzerfunktionen für Adobe Workfront Planning als eigenständiges Produkt beschrieben.

## Workfront Planning Packages

Ihr Unternehmen kann eines der folgenden eigenständigen Workfront Planning-Pakete erwerben:

* Prime
* Auswählen
* Ultimate

In der folgenden Tabelle wird beschrieben, was in den einzelnen Paketen enthalten ist:

| Planungsfähigkeit | Auswählen | Prime | Ultimate |
|---|---|---|---|
| Planung, Orchestrierung, Kampagnenverwaltung | ✓ | ✓ | ✓ |
| Unbegrenzte Arbeitsbereiche | ✓ | ✓ | ✓ |
| Datensätze pro Arbeitsbereich | 25,000 | 500,000 | 1,000,000 |
| Datensätze insgesamt (alle Arbeitsbereiche) | 500,000 | 2,000,000 | Unbegrenzt |
| Globale Datensatztypen und arbeitsbereichsübergreifende Verbindungen | – | ✓ | ✓ |
| Zugriff auf zukünftige Funktionen bei der Veröffentlichung | etwas | etwas | Most |

## Verfügbarkeit des Workfront-Planungspakets

<!--
the bullets repeat in the "Planning overview" article
-->

Workfront Planning ist verfügbar, wenn Ihr Unternehmen eines der folgenden Workfront-Pakete kauft:

* Workfront Workflow und Workfront Planning gemeinsam erworben. Jeder Benutzer in der Organisation verfügt über einen Workflow und eine Planning-Lizenz. Dadurch erhalten alle Benutzenden für beide Module vollständigen Zugriff auf alle Funktionen von Workfront.

* Workfront-Workflow für alle Personen in Ihrem Unternehmen und Workfront-Planung nur für einige Personen in Ihrem Unternehmen. Dadurch erhalten Benutzer vollen Zugriff auf alle Funktionen des Workflows und Benutzer, denen eine Planning-Lizenz zugewiesen ist, haben eingeschränkteren Zugriff auf Planungsfunktionen.

* Workfront Planning als eigenständiges Produkt für Benutzende in Ihrem Unternehmen. Dadurch erhalten Benutzerinnen und Benutzer keinen Zugriff auf Workfront Workflow-Funktionen und Planungsfunktionen.

Informationen zu den in Planning als eigenständiges Produkt enthaltenen Funktionen finden Sie im Abschnitt „In Workfront Planning als eigenständiges Produkt enthaltene Funktionen“ im Artikel [Erste Schritte mit Adobe Workfront Planning als eigenständiges Produkt](/help/quicksilver/planning/planning-sta/planning-sta-overview.md).

## Benutzerlizenz und Zugriffsebenen

<!-- should this be moved to the Manage users in Planning article??-->

Planungslizenzen sind Teil der Zugriffsebene, die Benutzern zugewiesen wird.

Für Planning als eigenständiges Produkt können Sie den Zugriffsebenen keine Lizenzen zuweisen. Sie sind bereits hartcodiert, um in die den Benutzern zugewiesenen Zugriffsebenen aufgenommen zu werden.

Je nachdem, mit welcher Workfront Planning-Paketkombination Ihr Unternehmen Einkäufe tätigt, haben Sie möglicherweise eine der folgenden Zugriffsebenen für Ihre Benutzerinnen und Benutzer:

* **Planning-Administrator**: Verfügbar für Workfront Planning-Pakete. Planning-Administratoren werden automatisch zugewiesen, wenn Benutzende zum Admin Console hinzugefügt werden.
* **Planning Standard**: Verfügbar für alle Workfront Planning-Pakete. Sie können diese Zugriffsebenen Benutzern zuweisen, wenn Sie sie erstellen.

### Übersicht über die Planning-Administratornavigation

<!--

Ein Benutzer mit der Zugriffsebene „Planning-Administrator“ hat die folgenden Funktionen:

* Hat vollständigen Verwaltungszugriff auf das System.
* Kann Benutzer erstellen und bearbeiten, Teams verwalten, benutzerdefinierte Quartale konfigurieren und auf alle Setup-Unterseiten zugreifen.

  Weitere Informationen finden Sie unter:

   * [Verwalten von Benutzern in Adobe Workfront Planning als eigenständiges Produkt](/help/quicksilver/planning/planning-sta/manage-users-in-planning-sta.md)
   * [Verwalten von Teams in Adobe Workfront Planning als eigenständiges Produkt](/help/quicksilver/planning/planning-sta/manage-teams-in-planning-sta.md)
* Kann Anfragen senden und verwalten.

  Weitere Informationen finden Sie unter [Adobe Workfront Planning Requests: Article index](/help/quicksilver/planning/requests/requests-article-index.md).
* Weist im Hauptmenü die folgenden Bereiche auf:

   * **Planning**: Bietet alle Funktionen für Planning-Objekte, um sie zu erstellen, zu löschen, freizugeben und zu verbinden.
   * **Benutzer**: Sie können Benutzer hinzufügen und ihre Profile bearbeiten.
   * **Anfragen**
   * **Setup**
* Enthält die folgenden Abschnitte im Bereich Setup :

   * **Teams**: Sie können Teams hinzufügen, entfernen oder bearbeiten. Die Bearbeitung ist auf den Team-Namen, die Beschreibung und die Mitglieder beschränkt. Es sind keine Filter-, Ansicht-, Gruppierungs- oder Exportsteuerelemente verfügbar.
   * **Anmelden als**: Stellvertretend für andere Benutzende agieren, um Fehler zu beheben.
   * **Benutzerdefinierte Quartale**: Konfigurieren Sie benutzerdefinierte Geschäftsquartale, die in den Ansichten „Planung - Zeitleiste“ angezeigt werden.
   * System

* Verfügt über die folgenden Abschnitte im Bereich System :

   * **Kundeninformationen**: Details zu Kunden und Organisationen anzeigen.
   * **Voreinstellungen**: Überprüfen und konfigurieren Sie die Voreinstellungen auf Systemebene.

### Übersicht über die Standardnavigation in Planning

Ein Benutzer mit der Zugriffsebene Planning Standard verfügt über die folgenden Funktionen:

* Kann Arbeitsbereiche und deren Inhalte verwalten.
* Kann Anfragen senden und verwalten.

  Weitere Informationen finden Sie unter [Adobe Workfront Planning Requests: Article index](/help/quicksilver/planning/requests/requests-article-index.md).
* Planning Standard-Benutzer können im Hauptmenü auf die folgenden Bereiche zugreifen:

   * **Planung**
   * **Benutzer**: Sie haben schreibgeschützten Zugriff auf Benutzer. Sie können keine Benutzer erstellen oder bearbeiten. <!--not sure if this is still true-->
   * **Anfragen**

* Sie haben keinen Zugriff auf Setup oder seine Abschnitte.

## Zugriffsebenen für Planning als eigenständiges Produkt konfigurieren

Zugriffsebenen sind integriert, wenn Sie Planning als eigenständiges Produkt erwerben, und Sie können sie nicht für Planning-Benutzer konfigurieren.

So weisen Sie Benutzern Zugriffsebenen zu:

* Erstellen Sie Benutzende in der Adobe-Konsole.

  Die folgenden Szenarien sind vorhanden:

   * Benutzende, die der Adobe-Konsole als Admins hinzugefügt wurden, erhalten in Workfront Planning die Zugriffsebene eines Planning-Administrators.
   * Benutzende, die der Adobe-Konsole hinzugefügt wurden, da ihnen in Workfront Planning eine Planning Standard-Zugriffsebene zugewiesen werden kann. Dies ist der einzige Zugriff, der neuen Benutzenden in Workfront Planning als eigenständiges Produkt zugewiesen werden kann.

Weitere Informationen finden Sie unter [Benutzer verwalten](/help/quicksilver/planning/planning-sta/manage-users-in-planning-sta.md).

## Gewähren von Berechtigungen in Workfront Planning als eigenständiges Produkt

Sie können die folgenden Objekte in Workfront Planning als eigenständiges Produkt freigeben:

* Arbeitsbereiche
* Eintragstypen
* Einträge
* Ansichten

Die Freigabe von Objekten in Planning als eigenständiges Produkt ist identisch mit der Freigabe in Planning, wenn sie zusammen mit einem Workflow-Paket gekauft wird.

Weitere Informationen finden Sie unter [Adobe Workfront Planning-Zugriffsinformationen: Artikelindex](/help/quicksilver/planning/access/access-information.md).

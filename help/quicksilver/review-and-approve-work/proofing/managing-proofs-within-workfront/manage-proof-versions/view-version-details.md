## Details einer früheren Testversion anzeigen und verwalten

1. Bewegen Sie in einer Dokumentliste den Mauszeiger über die Zeile, die den Testversand enthält, und klicken Sie auf **Dokumentdetails**.
1. Klicken Sie oben auf der Seite &quot;Dokumentdetails&quot;auf das Dropdown-Menü neben dem Namen und klicken Sie dann auf den Namen der Version, die Sie anzeigen und verwalten möchten.

   ![](assets/version-drop-dn-doc-dtls-nwe-350x93.png)

   Neben der Anzeige der Details der Version können Sie Änderungen an der Version vornehmen, z. B. ihren Namen, Metadaten und Testversandeinstellungen (sofern es sich um einen Dokumentversand handelt).

## Anzeigen der Testversanddetails für eine frühere Version

Benutzer müssen über eine Testlizenz verfügen, um die Testversanddetails einer früheren Version eines Testdokuments anzeigen zu können.

1. Wechseln Sie zum Projekt, zur Aufgabe oder zum Problem, das/das das Dokument enthält, und wählen Sie dann **Dokumente**.
1. Suchen Sie den gewünschten Beweis.
1. Im **Version** in der Zusammenfassung klicken Sie auf die Version und anschließend auf **Details** in der Dropdown-Liste, die angezeigt wird.

1. Klicken Sie auf der Seite &quot;Dokumentdetails&quot;auf **Testversand-Workflow** im linken Bereich, um einen der folgenden Schritte auszuführen:

   * Fügen Sie einen automatisierten Workflow hinzu. Weitere Informationen finden Sie im Abschnitt im Artikel .
   * Geben Sie die öffentliche URL des Testversands frei. Weitere Informationen finden Sie unter [Testversand-Link freigeben](../../../../review-and-approve-work/proofing/managing-proofs-within-workfront/share-a-proof-in-workfront.md#share) in [Freigeben eines Testversands in Adobe Workfront](../../../../review-and-approve-work/proofing/managing-proofs-within-workfront/share-a-proof-in-workfront.md).
   * Zeigen Sie alle Aktivitäten an, die beim Testversand aufgetreten sind.
   * Senden Sie beim Testversand Erinnerungsnachrichten an Validierungsverantwortliche.

1. Klicken **Fertig**.

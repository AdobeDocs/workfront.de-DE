# Zugriff [!DNL Adobe Workfront] Objekte aus einem freigegebenen Link in [!DNL Slack]

Wenn Links zu [!DNL Adobe Workfront] -Objekte für Sie in einer [!DNL Slack] -Kanal können Sie zusätzliche Informationen zu ihnen in [!DNL Slack]nach der Installation und Konfiguration [!DNL Workfront for Slack].

Weitere Informationen zur Konfiguration [!DNL Workfront for Slack], siehe [Konfigurieren [!DNL Adobe Workfront for Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/configure-workfront-for-slack.md).

Sie müssen bei [!DNL Workfront] in einer separaten Browser-Registerkarte, um den Link zu öffnen oder zusätzliche Aktionen für diese Objekte von durchzuführen [!DNL Slack].

Wenn ein Benutzer einen Link zu einem Objekt in [!DNL Workfront] bei Ihnen werden möglicherweise auch zusätzliche Informationen zum Objekt zusammen mit dem Link angezeigt. Der Benutzer, der den Link sendet, muss mindestens über [!UICONTROL Ansicht] Berechtigungen für das Objekt, damit zusätzliche Informationen in [!DNL Slack].

## Zugriffsanforderungen

Sie müssen über den folgenden Zugriff verfügen, um die in diesem Artikel beschriebene Funktion verwenden zu können:

## Voraussetzungen

Bevor Sie auf Ihre Favoriten und aktuellen Artikel zugreifen können, wählen Sie [!DNL Slack], müssen Sie

* Konfigurieren [!DNL Workfront for Slack]\
   Anweisungen zum Konfigurieren von [!DNL Workfront for Slack], siehe [Konfigurieren [!DNL Adobe Workfront for Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/configure-workfront-for-slack.md).

## Zugreifen auf Projekte über [!DNL Slack]

1. Melden Sie sich bei Ihrer [!DNL Slack] -Instanz und melden Sie sich bei [!DNL Workfront from Slack.]

   Weitere Informationen zur Anmeldung bei [!DNL Workfront] von [!DNL Slack], siehe &quot;Anmelden bei [!DNL Workfront from Slack]&quot; Abschnitt in [Zugriff [!DNL Adobe Workfront from Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navigieren Sie zu einer Nachricht mit einem Link zu einer [!DNL Workfront] Projekt.

   >[!NOTE]
   >
   >Der Benutzer, der den Link freigegeben hat, muss mindestens über [!UICONTROL Ansicht] Berechtigungen für das Projekt, damit Sie zusätzliche Informationen dazu sehen können.

   ![slack_project_details.png](assets/slack-project-details-350x199.png)

1. (Optional) Klicken Sie auf den Namen des Projekts, um es in [!DNL Workfront] in einer neuen Browser-Registerkarte.
1. Überprüfen Sie die folgenden zusätzlichen Informationen zum Projekt:

   * **[!UICONTROL Status]:** Weitere Informationen zum Projektstatus finden Sie unter [Zugriff auf die Liste der Systemprojektstatus](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/project-statuses.md).

   * **[!UICONTROL Geplantes Abschlussdatum]**: Weitere Informationen zum geplanten Abschlussdatum finden Sie unter [Festlegen des geplanten Abschlussdatums des Projekts](../../manage-work/projects/planning-a-project/project-planned-completion-date.md).

   * **[!UICONTROL Portfolio]** Name: Weitere Informationen zu Portfolios finden Sie unter [Übersicht über Portfolio in Adobe Workfront](../../manage-work/portfolios/portfolios-overview/portfolio-overview.md).

   * **[!UICONTROL Bedingung]**: Weitere Informationen zu Projektbedingungen finden Sie unter [Übersicht über Projektbedingung und Bedingungstyp](../../manage-work/projects/manage-projects/project-condition-and-condition-type.md).

   * **[!UICONTROL Prozent abgeschlossen]**: Weitere Informationen zum Wert &quot;Prozent abgeschlossen&quot;eines Projekts finden Sie unter [Projekt-Prozent Vollständige Übersicht](../../manage-work/tasks/task-information/project-percent-complete.md).

   * **[!UICONTROL Referenz Nr.]**: Weitere Informationen zu Objektreferenznummern finden Sie unter [Referenzanzahl von Objekten verwenden](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Optional) Klicken Sie auf **[!UICONTROL Abonnieren]** , um das Projekt zu abonnieren.

   Sie erhalten eine Bestätigung, dass Sie das Projekt erfolgreich abonniert haben.

## Zugriff auf Aufgaben über [!DNL Slack]

1. Melden Sie sich bei Ihrer [!DNL Slack] -Instanz und melden Sie sich bei [!DNL Workfront from Slack].

   Weitere Informationen zur Anmeldung bei [!DNL Workfront] von [!DNL Slack], siehe &quot;Anmelden bei [!DNL Workfront] von [!DNL Slack]&quot; Abschnitt in [Zugriff [!DNL Adobe Workfront from Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navigieren Sie zu einer Nachricht mit einem Link zu einer [!DNL Workfront] Aufgabe.

   >[!NOTE]
   >
   >Der Benutzer, der den Link freigegeben hat, muss mindestens über [!UICONTROL Ansicht] Berechtigungen für die Aufgabe, damit Sie zusätzliche Informationen dazu sehen können.

1. (Optional) Klicken Sie auf den Namen der Aufgabe, um sie in [!DNL Workfront] in einer neuen Browser-Registerkarte.
1. Überprüfen Sie die folgenden zusätzlichen Informationen zur Aufgabe:

   * **[!UICONTROL Status]**

      Weitere Informationen zum Aufgabenstatus finden Sie unter [Zugriff auf die Liste der Systemaufgabenstatus](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/task-statuses.md).

   * **[!UICONTROL Prozent abgeschlossen]**
   * **[!UICONTROL Zugeordneter Name]**
   * **[!DNL Planned Completion Date]**
   * **[!UICONTROL Projektname]** oder **[!UICONTROL Übergeordneter Aufgabenname]**
   * **[!UICONTROL Referenznummer]**

      Weitere Informationen zu Objektreferenznummern finden Sie unter [Referenzanzahl von Objekten verwenden](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Optional) Klicken Sie auf **[!UICONTROL Abonnieren]** , um die Aufgabe zu abonnieren.

   Sie erhalten eine Bestätigung, dass Sie die Aufgabe erfolgreich abonniert haben.

1. (Optional) Klicken Sie auf **[!UICONTROL Dem Benutzer zuweisen]** , um die Aufgabe einem Benutzer zuzuweisen.
1. (Bedingt) Beginnen Sie mit der Eingabe des Benutzernamens, wenn Sie auf **[!UICONTROL Dem Benutzer zuweisen]**.

   Sie können eine Aufgabe nicht einer Auftrags- oder Team-Rolle zuweisen von [!DNL Slack].

## Zugriffsprobleme über [!DNL Slack]

1. Melden Sie sich bei Ihrer [!DNL Slack] -Instanz und melden Sie sich bei [!DNL Workfront] von [!DNL Slack].

   Weitere Informationen zur Anmeldung bei Workfront von [!DNL Slack], siehe &quot;Anmelden bei [!DNL Workfront] von [!DNL Slack]&quot; Abschnitt in [Zugriff [!DNL Adobe Workfront] von [!DNL Slack]](../../workfront-integrations-and-apps/using-workfront-with-slack/access-workfront-from-slack.md).

1. Navigieren Sie zu einer Nachricht mit einem Link zu einer [!DNL Workfront] Problem.

   >[!NOTE]
   >
   >Der Benutzer, der den Link freigegeben hat, muss mindestens über [!UICONTROL Ansicht] Berechtigungen auf das Problem zuweisen, damit Sie zusätzliche Informationen dazu erhalten.

1. (Optional) Klicken Sie auf den Namen des Problems, um es in [!DNL Workfront] in einer neuen Browser-Registerkarte.
1. Überprüfen Sie die folgenden zusätzlichen Informationen zur Aufgabe:

   * **[!UICONTROL Status]**: Weitere Informationen zum Status von Problemen finden Sie unter [Zugriff auf die Liste der Systemfehlerstatus](../../administration-and-setup/customize-workfront/creating-custom-status-and-priority-labels/issue-statuses.md).

   * **[!UICONTROL Zugeordneter Name]**
   * **[!UICONTROL Geplantes Abschlussdatum]**
   * **[!UICONTROL Projektname]**
   * **[!UICONTROL Referenz Nr.]**: Weitere Informationen zu Objektreferenznummern finden Sie unter [Referenzanzahl von Objekten verwenden](../../workfront-basics/navigate-workfront/search/reference-number-of-objects.md).

1. (Optional) Klicken Sie auf **[!UICONTROL Abonnieren]** um sich für das Problem zu registrieren.

   Sie erhalten eine Bestätigung, dass Sie das Problem erfolgreich abonniert haben.

1. (Optional) Klicken Sie auf **[!UICONTROL Dem Benutzer zuweisen]** , um das Problem einem Benutzer zuzuweisen.
1. (Bedingt) Beginnen Sie mit der Eingabe des Benutzernamens, wenn Sie auf **[!UICONTROL Dem Benutzer zuweisen]**.

   Sie können ein Problem nicht einer Auftrags- oder Team-Rolle zuweisen von [!DNL Slack].

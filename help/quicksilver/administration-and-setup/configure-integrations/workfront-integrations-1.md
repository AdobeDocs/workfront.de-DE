# [!DNL Adobe Workfront]-Integrationen

<!--Audited: 12/2023-->

Sie können [!DNL Adobe Workfront] mit Anwendungen von Drittanbietern sowie anderen [!DNL Adobe]-Produkten integrieren. Integrationen können den Nutzen von Workfront erweitern und an die Anforderungen Ihrer Organisation angepasst werden.

Die Integrationen für [!DNL Workfront] lassen sich in die folgenden Kategorien einteilen:

## Native Integrationen

[!DNL Workfront] bietet verschiedene Integrationen, die Sie entweder direkt über Workfront oder direkt über eine andere Anwendung konfigurieren können, indem Sie das [!DNL Workfront]-Add-in für diese Anwendung installieren.

Einige dieser Integrationen sind ohne zusätzliche Kosten verfügbar, während für andere Integrationen ein zusätzlicher Kauf erforderlich ist:

* [Erste Schritte mit nativen Integrationen](#get-started-with-built-in-native-integrations)
* [Ohne zusätzliche Kosten verfügbare Integrationen](#integrations-available-at-no-additional-cost)
* [Integrationen, für die ein zusätzlicher Kauf erforderlich ist](#integrations-requiring-an-additional-purchase)

### Erste Schritte mit nativen Integrationen {#get-started-with-built-in-native-integrations}

Verwenden Sie den folgenden Workflow, um mit der Nutzung nativer Integrationen zu beginnen:

1. **Festlegen, welche nativen Integrationen Ihren Anforderungen am besten entsprechen**

   Bei der Auswahl einer Integration ist es wichtig, eine Anwendung auszuwählen, die bereits regelmäßig verwendet wird. Wird [!DNL Workfront] mit einer regelmäßig oder häufig verwendeten Anwendung integriert, steigert dies die Akzeptanz bei Ihren Benutzerinnen und Benutzern.

   Eine Liste der verfügbaren Integrationen finden Sie unter [Ohne zusätzliche Kosten verfügbare Integrationen](#integrations-available-at-no-additional-cost) und [Integrationen, für die ein zusätzlicher Kauf erforderlich ist](#integrations-requiring-an-additional-purchase).

1. **Ermitteln Ihres Technologie-Admins**

   Ermitteln Sie den bzw. die Technologie-Admin Ihrer Organisation für die Anwendung, die Sie mit [!DNL Workfront] integrieren möchten. Wenn Sie sich beispielsweise für eine Integration mit [!DNL Slack] entscheiden, müssen Sie Kontakt mit dem bzw. der [!DNL Slack]-Admin Ihres Unternehmens aufnehmen und ihn bzw. sie in alle Konfigurationsanforderungen innerhalb seiner bzw. ihrer Plattform involvieren. Sie sollten diese bzw. diesen Admin auch in jedes Gespräch mit dem Kunden-Support von [!DNL Workfront] einbeziehen.

   Damit die oder der Admin mit der Arbeit beginnen kann, müssen Sie ihr bzw. ihm die spezifischen Anweisungen für diese Integration bereitstellen. Links zu diesen Anweisungen finden Sie unter [Ohne zusätzliche Kosten verfügbare Integrationen](#integrations-available-at-no-additional-cost).

1. **Kontakt mit dem Kunden-Support von [!DNL Workfront] herstellen**

   Wenn der bzw. die Admin Probleme mit dem Setup der nativen Integration hat, können Sie sich an den Kunden-Support von [!DNL Workfront] wenden, um eine Anleitung oder Informationen zur Fehlerbehebung zu erhalten. Zwar richtet [!DNL Workfront] die Integration nicht für Sie ein, wir sind aber bestrebt, Ihnen alle erforderlichen Tools und Ressourcen zur Verfügung zu stellen, die Sie für die Selbsthilfe benötigen. Sie können direkt mit dem Support sprechen oder die umfangreichen Ressourcenartikel lesen, die Sie bei der einfachen Einrichtung Ihrer Integration unterstützen.

   Informationen dazu, wie Sie Kontakt zum Kunden-Support herstellen können, finden Sie unter [Kontaktieren des Kunden-Supports](../../workfront-basics/tips-tricks-and-troubleshooting/contact-customer-support.md).

   Links zu Hilfeartikeln finden Sie unter [Ohne zusätzliche Kosten verfügbare Integrationen](#integrations-available-at-no-additional-cost) und [Integrationen, die einen zusätzlichen Kauf erfordern](#integrations-requiring-an-additional-purchase).

   Anweisungen zum Erstellen eines Support-Tickets finden Sie unter [Kontaktieren des Kunden-Supports](../../workfront-basics/tips-tricks-and-troubleshooting/contact-customer-support.md).

   >[!NOTE]
   >
   >Sie müssen der zugewiesene Support-Kontakt in Ihrer Organisation sein, um ein Support-Ticket einreichen zu können. Nachdem das Ticket eingereicht wurde, können Sie Ihren bzw. Ihre Technologie-Admin für das andere System zum Gespräch hinzufügen. Er bzw. sie kann den Support nicht direkt kontaktieren.

Wir wissen, dass die verfügbaren nativen Integrationen möglicherweise nicht alle Anforderungen Ihres Unternehmens erfüllen. Wenden Sie sich in diesem Fall an Ihre Kundenbetreuung, um eine benutzerdefinierte Integration zu erörtern.

### Integrationen, für die ein zusätzlicher Kauf erforderlich ist {#integrations-requiring-an-additional-purchase}

In [!DNL Workfront] können Sie gegen Aufpreis andere native Integrationen und integrierte Produkte nutzen. Einzelheiten zu den Preisen erfahren Sie bei Ihrer Kundenbetreuung.

* [!DNL Workfront Fusion]

  [!DNL Adobe Workfront Fusion] verknüpft Aktionen innerhalb und zwischen Anwendungen und Services, um ein Szenario zu erstellen, in dem Ihre Daten automatisch übertragen und umgewandelt werden. Es verfügt über dedizierte Connectoren für viele Anwendungen, kann jedoch über eine öffentliche API mit jeder Anwendung verbunden werden.

  Weitere Informationen finden Sie unter [[!DNL Adobe Workfront Fusion] Überblick](https://experienceleague.adobe.com/de/docs/workfront-fusion/using/get-started-with-fusion/understand-workfront-fusion/workfront-fusion-overview).

* [!DNL Adobe Experience Manager Assets]
* [!DNL Anaplan] (erfordert [!DNL Workfront Fusion])

Durch die Integration zwischen Workfront und Experience Manager Assets oder Assets Essentials können Organisationen Content Velocity und Time-to-Market beschleunigen, indem sie die Arbeitsabläufe und die Verwaltung digitaler Assets eng miteinander verknüpfen.

Weitere Informationen finden Sie unter [Überblick über Adobe Workfront für Experience Manager Assets-Integrationen](../../documents/adobe-workfront-for-experience-manager-assets-essentials/aem-asset-integrations.md).

## Partnerintegrationen

Workfront verfügt über viele Partnerintegrationen, die Ihr Workfront-Erlebnis bereichern. Beispiele für vorhandene Partnerangebote sind Integrationen mit Claravine, RWS Group und mehr.

Weitere Informationen zu diesen und anderen Partnerintegrationen finden Sie unter [Exchange Marketplace](https://exchange.adobe.com/apps/browse/ec?page=1&partnerLevel=All&product=WRKFRNT&sort=RELEVANCE).

## [!DNL Workfront]-API

Mit der [!DNL Workfront]-API können Sie Ihr [!DNL Workfront]-Erlebnis mithilfe unserer öffentlichen API erweitern und verbessern. Sie können Ihre eigenen Integrationen mit unserer offenen API erstellen.

Eine detaillierte Liste der in der [!DNL Workfront]-API verfügbaren Objekte finden Sie im [!DNL Workfront] [API-Explorer](../../wf-api/general/api-explorer.md).

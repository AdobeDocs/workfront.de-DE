## Adobe Workfront-Boards sind jetzt verfügbar!

Pinnwände sind flexible Tools, die eine Zusammenarbeit zwischen Teams ermöglichen, indem sie Zugriff auf eine freigegebene Pinnwand mit Spalten und Karten bieten.

Pinnwände bieten folgende Möglichkeiten:

* Schnelles Einrichten einer Task-Pinnwand mit mehreren Spalten
* Spalten so konfigurieren, dass ein Status oder eine Kategorie angezeigt wird
* Fügen Sie andere Benutzer zur Pinnwand hinzu und weisen Sie sie Karten zu
* Schnelles Hinzufügen von offenen Karten und Checklisten

Beachten Sie, dass die Karten auf einer Pinnwand nicht mit Objekten und Arbeitselementen in Adobe Workfront verbunden sind.

Ein Systemadministrator muss Pinnwände in Layout-Vorlagen aktivieren, damit die Option allen Benutzern im Hauptmenü zur Verfügung steht.

Weitere Informationen finden Sie unter [Pinnwände - Übersicht](../../../agile/boards-overview.md).

## Zusätzliche Verbesserungen an Workfront-Motherboards

Die folgenden zusätzlichen Verbesserungen sind jetzt für Workfront-Boards verfügbar:

* Tagging-Karten auf Pinnwänden

  Sie können jetzt Karten auf Ihrer Pinnwand mit farbcodierten Tags kategorisieren. Mit Tags können Sie Karten schnell identifizieren. Sie können Ihre Pinnwand auch nach angewendeten Tags sortieren.

* Verwalten von Karten auf Pinnwänden

  Die folgenden Funktionen wurden hinzugefügt, um Ihnen bei der Verwaltung von Karten auf Ihrer Pinnwand zu helfen:

   * Karte kopieren: Eine vorhandene Karte auf der Pinnwand kopieren.
   * Karte verschieben: Mit den neuen Menüoptionen Spaltenoberseite und Spaltenunterseite können Sie Karten schnell an den oberen oder unteren Rand einer Pinnwand verschieben.

* In Pinnwänden suchen

  Wir haben eine Suchleiste hinzugefügt, mit der Sie alle Karten auf Ihrem Board durchsuchen können.

* Festlegen eines Fälligkeitsdatums für eine Karte in Pinnwänden

  Sie können jetzt ein Fälligkeitsdatum für einzelne Karten auf Ihrer Pinnwand festlegen.

Weitere Informationen finden Sie unter [Erste Schritte mit Pinnwänden in Adobe Workfront](../../../agile/get-started-with-boards/get-started-with-boards.md).

## Option zum Rückgängigmachen von Aktualisierungsposten

Es ist jetzt einfacher, Fehler beim Posten einer Aktualisierung zu erkennen. Wenn Sie einen Kommentar auf der Registerkarte Aktualisieren eines Objekts abschließen, wird jetzt 7 Sekunden lang ein Popup-Fenster erstellt, in dem Sie den Beitrag abbrechen und zur Bearbeitung zurückkehren können, bevor das System ihn mit einem Zeitstempel versehen oder E-Mails und In-App-Benachrichtigungen sendet. Wenn Sie das Popup-Fenster schließen, die Seite verlassen oder 7 Sekunden warten, bis das Fenster beendet ist, wird der Beitrag normal ausgeführt.

Weitere Informationen finden Sie unter [Arbeit aktualisieren](../../../workfront-basics/updating-work-items-and-viewing-updates/update-work.md).

## Aktualisiertes Erlebnis beim Kopieren und Verschieben von Problemen

Damit Ihre Nutzung von Workfront mit dem neuen Adobe Workfront-Erlebnis konsistent ist, haben wir die Benutzeroberfläche zum Kopieren und Verschieben von Problemen neu gestaltet. Dies ist derzeit verfügbar, wenn ein einzelnes Problem kopiert oder verschoben oder Probleme stapelweise aus einer Liste oder einem Bericht kopiert oder verschoben werden.

Zu den Verbesserungen dieser neu gestalteten Benutzeroberfläche gehören unter anderem:

* Alle Informationen, die Sie aktualisieren müssen, bevor der Verschiebevorgang auf einer fortlaufenden Seite angezeigt wird.
* Workfront prüft, ob Sie unmittelbar nach Auswahl des Projekts Zugriff auf das Zielprojekt haben. Vor dieser Verbesserung hat Workfront Sie gewarnt, dass Sie nicht über den richtigen Zugriff verfügen, nachdem Sie die Verschiebung bestätigt haben, die dazu führte, dass zusätzliche Schritte durchgeführt wurden und die Verschiebung nicht zulässig war.
* Möglichkeit, Zugriff für ein Projekt anzufordern, in das Sie die Probleme verschieben möchten, ohne das Feld „Aufgabe verschieben“ zu verlassen.
* Möglichkeit, Elemente (Zuweisungen, Fortschritt, Dokumente, Berechtigungen, Aktualisierungen) aus einem Problem zu entfernen, wenn Sie es an einen anderen Speicherort verschieben. Diese Funktion war zuvor nur zum Kopieren von Problemen verfügbar.
* Möglichkeit, beim Kopieren eines Problems zusätzlich zur Auswahl eines Zielprojekts eine Zielaufgabe auszuwählen.

Weitere Informationen zum Verschieben oder Kopieren von Problemen finden Sie in den folgenden Artikeln:

* [Probleme kopieren](../../../manage-work/issues/manage-issues/copy-issues.md)
* [Probleme verschieben](../../../manage-work/issues/manage-issues/move-issues.md)

## Neues Erlebnis beim Kopieren eines Projekts

Damit Ihre Verwendung von Workfront mit dem neuen Adobe Workfront-Erlebnis konsistent ist, haben wir die Benutzeroberfläche zum Kopieren von Projekten neu gestaltet. Dies ist derzeit beim Kopieren eines Projekts von der Projektseite oder beim Kopieren eines Projekts aus einer Liste oder einem Bericht verfügbar. Vor diesem Update konnten Sie nur ein Projekt von der Projektseite kopieren.

Weitere Informationen finden Sie unter [Kopieren eines Projekts](../../../manage-work/projects/manage-projects/copy-project.md).

## Benutzer nach der Konvertierung eines Problems in ein Projekt im Dashboard, in der Liste oder im Bericht belassen

Um die Effizienz zu steigern und die Anzahl der Klicks zu reduzieren, haben wir eine Verbesserung beim Konvertieren von Problemen in Projekte aus einer Liste, einem Bericht oder einem Dashboard veröffentlicht.

Benutzer bleiben auf der Liste, dem Bericht oder einem Dashboard, nachdem sie ein Problem in ein Projekt konvertiert haben, anstatt zur Projektseite weitergeleitet zu werden. Nach Abschluss der Konvertierung wird eine Erfolgsmeldung mit dem Link zum Projekt angezeigt, damit Sie bei Bedarf einfach zum Projekt navigieren können.

Weitere Informationen finden Sie unter [Problem in ein Projekt konvertieren](../../../manage-work/issues/convert-issues/convert-issue-to-project.md).

## Zuordnungsstunden werden bei Änderungen an Zuweisungen nicht mehr entfernt

>[!NOTE]
>
>Diese Funktion sollte ursprünglich mit Version 22.2 veröffentlicht werden. Es wird am 21. April 2022 in der Produktionsumgebung veröffentlicht.

Um die Genauigkeit Ihrer Daten zu gewährleisten, haben wir eine Änderung vorgenommen, um die Zuordnungsstunden beizubehalten und die geplanten Stunden für Aufgaben unverändert zu lassen, wenn Änderungen an Zuweisungen für die Aufgabe vorgenommen werden.

Die folgenden Änderungen wurden an Aufgaben mit dem Typ Einfache Dauer vorgenommen:

* Geplante Stunden werden beibehalten, wenn alle Zugewiesenen entfernt werden.
* Einzelne Zuweisungen bleiben beim Ersetzen von Benutzern und Rollen erhalten.
* Einzelne Zuweisungen bleiben bei der Rolle erhalten, wenn Benutzer entfernt werden. (Aus Version entfernt. Die geplanten Stunden werden nun auf 0 gesetzt, nachdem alle Bevollmächtigten entfernt wurden.)

Weitere Informationen zu den geplanten Stunden finden Sie unter [Übersicht über die geplanten Stunden](../../../manage-work/tasks/task-information/planned-hours.md).

## Freigeben von Ordnern nur in den obersten fünf Ebenen einer Ordnerhierarchie

>[!NOTE]
>
>Diese Funktion ist vorübergehend nicht verfügbar. Wir werden diese Versionshinweise aktualisieren, wenn die Funktion in der Produktionsumgebung verfügbar ist.

Um eine optimale Leistung für Benutzer zu gewährleisten, die Ordner freigeben, beschränken wir die Freigabe derzeit auf die fünf obersten Ebenen in einer Ordnerhierarchie für ein Objekt.

Jeder Ordner auf der sechsten Ebene oder darunter erbt seine Freigabekonfigurationen von dem Ordner direkt darüber.

Weitere Informationen zum Freigeben von Ordnern finden Sie unter [Freigeben eines Dokumentordners](../../../workfront-basics/grant-and-request-access-to-objects/share-a-document-folder.md).

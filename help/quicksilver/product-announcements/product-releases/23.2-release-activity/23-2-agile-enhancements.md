# 23.2 Agile-Verbesserungen

Auf dieser Seite werden alle Agile-Verbesserungen beschrieben, die mit Version 23.2 an der Vorschau-Umgebung vorgenommen wurden. Diese Verbesserungen werden mit Version 23.2 in der Produktionsumgebung verfügbar gemacht.

Eine Liste aller Änderungen, die zu diesem Zeitpunkt im Versionszyklus 23.2 verfügbar sind, finden Sie unter Übersicht über Version [23.2](/help/quicksilver/product-announcements/product-releases/23.2-release-activity/23-2-release-overview.md).

## Hinzufügen von Aufgaben und Problemen aus Objektdetails zu einer Workfront-Pinnwand oder einem Arbeitsablauf

Sie können bestehende Aufgaben oder Probleme jetzt direkt über die Objektdetails zu einer Workfront-Pinnwand oder einer Workstream-Kartenliste hinzufügen. Alle Elemente, die Sie einer Pinnwand hinzufügen, werden zu verbundenen Karten, und alle Elemente, die zu einem Arbeitsablauf hinzugefügt werden, werden als ungeplante Karten zur Kartenliste hinzugefügt.

Weitere Informationen finden Sie unter [Hinzufügen vorhandener Aufgaben oder Probleme zu einer Pinnwand](/help/quicksilver/agile/get-started-with-boards/add-card-from-list-to-board.md).

## In Adobe Workfront-Motherboards verfügbare Iterationsfunktionen

Mehrere neue Funktionen, die in Workfront-Boards verfügbar sind, ermöglichen die Verwendung der Agile-Scrum-Funktion. Zu diesen Funktionen gehören:

* Arbeitsabläufe zum Gruppieren von Pinnwänden, die demselben Team zugeordnet sind und an der Arbeit zusammenarbeiten
* Eine Liste von Karten oder Arbeitsrückstand mit der Option, Quellen zu verwenden, um Karten mit Workfront-Aufgaben und -Problemen zu verbinden
* Pinnwände für Iterationsplanung und Iterationsprozess

Beachten Sie, dass Sammlungen in Arbeitsabläufe umbenannt wurden. Arbeitsabläufe helfen Ihnen, Daten auf unterschiedliche Weise zu visualisieren. Sie können Elemente auf Karten in einer Liste, auf einer Pinnwand oder in einer Iteration anzeigen. Karten in einem Arbeitsablauf können auch auf mehreren Pinnwänden gemeinsam genutzt werden. Mithilfe von Karten und Pinnwänden in einem Arbeitsablauf können Sie Workflows ganz einfach vereinfachen.

Weitere Informationen finden Sie unter [Verwalten von Arbeitsabläufen](/help/quicksilver/agile/use-boards-agile-planning-tools/manage-collections.md), [Erstellen einer Iteration in einem Arbeitsablauf](/help/quicksilver/agile/use-boards-agile-planning-tools/create-an-iteration-in-workstream.md) und [Verwenden der Kartenliste](/help/quicksilver/agile/use-boards-agile-planning-tools/use-card-list.md).

[Sehen Sie sich eine Videodemonstration dieser Funktion an](https://video.tv.adobe.com/v/3417059/){target=_blank}

## Aufgaben und Probleme aus Listen und Berichten zu einer Arbeitsablauf-Kartenliste hinzufügen

Sie können einem Arbeitsablauf in Workfront-Boards jetzt direkt aus einer Listen- oder Berichtsansicht vorhandene Aufgaben oder Probleme hinzufügen. Alle Elemente, die Sie dem Arbeitsablauf hinzufügen, werden als ungeplante Karten zur Kartenliste hinzugefügt.

Weitere Informationen finden Sie unter [Hinzufügen vorhandener Aufgaben oder Probleme zu einer Pinnwand](/help/quicksilver/agile/get-started-with-boards/add-card-from-list-to-board.md).

## Hinzufügen benutzerdefinierter Felder zu Karten auf einer Pinnwand

Sie können jetzt benutzerdefinierte Felder in Adobe Workfront-Pinnwände einfügen. Das Feld muss bereits in Workfront erstellt worden sein. Es ist nicht möglich, benutzerdefinierte Felder in einer Pinnwand zu entwerfen und neu zu erstellen.

Wie bei den Standardfeldern können Sie auch hier festlegen, dass das benutzerdefinierte Feld in der Vollansicht einer Karte und die verkürzte Ansicht auf der Pinnwand angezeigt werden soll.

Alle Daten aus dem benutzerdefinierten Feld auf der Karte sind schreibgeschützt.

Weitere Informationen finden Sie unter [Anpassen der auf einer Karte angezeigten Felder](/help/quicksilver/agile/get-started-with-boards/customize-fields-on-card.md).

## Hinzufügen von Aufgaben und Problemen aus Listen und Berichten zu einem Workfront-Board

Sie können einem Workfront-Board jetzt direkt über eine Listen- oder Berichtsansicht vorhandene Aufgaben oder Probleme hinzufügen. Alle Elemente, die Sie der Pinnwand hinzufügen, werden zu verbundenen Karten.

Außerdem kann das Feld Pinnwände jetzt zu Listen und Berichten für Aufgaben oder Probleme hinzugefügt werden. In diesem Feld werden alle Boards angezeigt, denen eine Aufgabe oder ein Problem hinzugefügt wurde.

Weitere Informationen finden Sie unter [Hinzufügen vorhandener Aufgaben oder Probleme zu einer Pinnwand](/help/quicksilver/agile/get-started-with-boards/add-card-from-list-to-board.md).


## Stunden auf verbundenen Karten auf einer Pinnwand protokollieren

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.

Sie können jetzt Stunden auf verbundenen Karten genauso protokollieren, wie Sie es bei einer Aufgabe oder einem Problem tun würden. Sie müssen über die richtigen Berechtigungen für die Aufgabe oder das Problem verfügen, um die Zeit protokollieren zu können.

Die Zeitprotokollierungsfelder werden nicht standardmäßig auf verbundenen Karten angezeigt. Sie müssen &quot;**&quot;** Bereich „Konfigurieren“ unter „Karten“ aktivieren.

Weitere Informationen finden Sie unter [Verwenden von Karten auf Pinnwänden](/help/quicksilver/agile/get-started-with-boards/connected-cards.md).


## Anpassen der Anzeige von Feldern auf einer Karte

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.


Mit der Anpassung können Sie jetzt konfigurieren, welche Felder auf einer Karte angezeigt werden - sowohl in der Vollansicht, wenn die Karte geöffnet ist, als auch in der reduzierten Kartenansicht auf der Pinnwand. Wenn Sie ein Feld deaktivieren, wird es in keiner der Ansichten angezeigt. Sie können auch ein Feld in der Vollansicht aktivieren und es in der verkürzten Ansicht ausblenden.

Weitere Informationen finden Sie unter [Anpassen der auf einer Karte angezeigten Felder](/help/quicksilver/agile/get-started-with-boards/customize-fields-on-card.md).

## Standardstatus für Karten definieren, die in eine Pinnwand verschoben werden

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.

Sie können jetzt einen Standardstatus festlegen, der auf Karten angewendet wird, die in eine bestimmte Spalte verschoben werden, indem Sie in den Spaltenrichtlinien einen benutzerdefinierten Status und einen Systemstatus auswählen. Wenn Sie eine Karte in die Spalte verschieben, versucht Workfront zunächst, den benutzerdefinierten Status anzuwenden (z. B. Warten auf Feedback). Wenn der benutzerdefinierte Status für diese Karte nicht verfügbar ist, wendet Workfront stattdessen den Systemstatus an (z. B. „Zurückgestellt„). Wenn der Status für die verbundene Aufgabe oder das verbundene Problem in den benutzerdefinierten oder Systemstatus geändert wird, der in der Spaltenrichtlinie festgelegt ist, wird die Karte automatisch in die Spalte verschoben.

Zuvor wurden Sie aufgefordert, einen Status für jede Karte auszuwählen, die in die Spalte verschoben wird, wenn mehrere Status verfügbar sind.

Weitere Informationen finden Sie unter [Spalten verwalten](/help/quicksilver/agile/get-started-with-boards/manage-board-columns.md).

[Sehen Sie sich eine Videodemonstration dieser Funktion an](https://video.tv.adobe.com/v/3415711/){target=_blank}

## Sammlungen jetzt in Adobe Workfront Boards verfügbar

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.

Sie können jetzt Sammlungen im Dashboard „Pinnwände“ erstellen. Eine Sammlung ist eine Gruppe von Boards für die Zusammenarbeit bei der Arbeit. Nachdem Sie die Sammlung benannt haben, können Sie der Sammlung Pinnwände mithilfe einer Reihe von Vorlagen hinzufügen, die vordefinierte Einstellungen wie Spaltennamen bieten. Sie können auch eigenständige Pinnwände in eine Sammlung verschieben. Sobald sich eine Pinnwand in einer Sammlung befindet, kann sie in eine andere Sammlung verschoben werden, sie kann jedoch nicht zu einer eigenständigen Pinnwand werden.

Das Hinzufügen von Membern zu einer Sammlung funktioniert auf die gleiche Weise wie das Hinzufügen von Membern zu einer Pinnwand. Eine Person oder ein Team muss Mitglied der Sammlung sein, bevor sie bzw. es als Mitglieder einem Board der Sammlung hinzugefügt werden kann.

Weitere Informationen finden Sie unter [Verwalten von Sammlungen](/help/quicksilver/agile/use-boards-agile-planning-tools/manage-collections.md).

[Sehen Sie sich eine Videodemonstration dieser Funktion an](https://video.tv.adobe.com/v/3415609/){target=_blank}

## Schätzfeld in verbundenen Karten, das dem Feld „Story-Punkte“ in Workfront-Objekten zugeordnet ist

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.

Das Schätzfeld auf verbundenen Karten in Workfront Boards ist jetzt dem Storypoints-Feld für das zugehörige Workfront-Objekt zugeordnet.

Das neue Feld Story-Punkte ist ein bearbeitbares Freiformfeld, das Sie einer Ansicht in einer Liste oder einem Bericht für Aufgaben oder Probleme hinzufügen können. Sie ist nicht an geplante Stunden oder Team-Arbeitsaufträge gebunden.

Zuvor war die Kartenschätzung ein manueller Eintrag und wurde keinem Feld in einer Aufgabe oder einem Problem zugeordnet.

Außerdem gibt es im Feld Schätzung sowohl auf Ad-hoc- als auch auf verbundenen Karten keine Zeichenbeschränkung mehr. Zuvor war die maximale Anzahl von Zeichen 99.

Weitere Informationen finden Sie unter [Verwenden von Karten auf Pinnwänden](/help/quicksilver/agile/get-started-with-boards/connected-cards.md).

## Vorschau der Karte in Aufnahmespalte

>[!NOTE]
>
>Diese Funktion ist nur über die frühzeitige Funktionsaktivierung für Workfront-Boards verfügbar.

Sie können jetzt in der Spalte Aufnahme auf eine verbundene Karte klicken, um eine schreibgeschützte Version ihres Inhalts anzuzeigen. Der Karteninhalt kann erst bearbeitet werden, wenn die Karte aus der Aufnahmespalte in eine andere Spalte auf der Pinnwand verschoben wird.

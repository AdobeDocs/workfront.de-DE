# Übersicht über Filter

<!-- Audited: 11/2024 -->

Sie können in Adobe Workfront Filter verwenden, um die Informationsmenge zu reduzieren, die auf dem Bildschirm in Listen, Berichten oder anderen Bereichen angezeigt wird.

>[!NOTE]
>
>In diesem Artikel werden alle Bereiche beschrieben, in denen Sie in Workfront Filter anwenden können, sowie allgemeine Informationen zu Filtern in Listen und Berichten. Informationen zu Filtern in bestimmten Bereichen außer Listen und Berichten finden Sie in den unten verlinkten Artikeln.

## Workfront-Filtertypen

In Workfront gibt es mehrere Filtertypen. Alle Filter erfüllen den gleichen Zweck, die Menge der Informationen zu begrenzen, die Sie auf dem Bildschirm anzeigen.

Im Folgenden finden Sie Filtertypen, die Sie in Workfront verwenden können:

  </tr>

## Temporäre Workfront-Filter

Mit Schnellfiltern können Sie mithilfe eines Keywords auf dem Bildschirm nach einem Element suchen. Nach dem Aktualisieren der Seite wird der temporäre Schnellfilter gelöscht. Temporäre Filter können nicht zur Wiederverwendung gespeichert werden.

Informationen zu Schnellfiltern finden Sie unter [Anwenden des Schnellfilters auf eine Liste](../../../workfront-basics/navigate-workfront/use-lists/apply-quick-filter-list.md).

## Permanente Workfront-Filter für Listen und Berichte

Sie können permanente Filter verwenden, die Sie, andere oder Workfront in Listen oder Berichten von Objekten erstellt haben.

Informationen zu Objekten in Workfront und dazu, für welche Sie Berichte erstellen können, finden Sie unter [Übersicht über Adobe Workfront-Objekte](../../../workfront-basics/navigate-workfront/workfront-navigation/understand-objects.md).

## Übersicht über Filter in Listen und Berichten

Beim Arbeiten mit Listen und Berichten können Sie Informationen auf dem Bildschirm mit den folgenden Filtertypen filtern:

* Integrierte Filter
* Neue Filter, die Sie oder andere von Grund auf erstellen
* Angepasste vorhandene Filter, die von anderen erstellt und für Sie freigegeben wurden
* Kopierte und bearbeitete Filter basierend auf vorhandenen Filtern

>[!IMPORTANT]
>
>Beim Erstellen oder Bearbeiten von Filtern in Berichten müssen Sie den Bericht bearbeiten, damit der neue Filter zum Standardfilter für den Bericht wird. Wenn Sie nur den Filter bearbeiten, wird außerhalb des Report Builders der Standardfilter des Berichts nicht aktualisiert.\
>Informationen zum Bearbeiten eines Berichts finden Sie im Artikel [Erstellen eines benutzerdefinierten Berichts](../../../reports-and-dashboards/reports/creating-and-managing-reports/create-custom-report.md).

Informationen zu Workfront-Listen finden Sie unter [Erste Schritte mit Listen in Adobe Workfront](../../../workfront-basics/navigate-workfront/use-lists/view-items-in-a-list.md).

Informationen zu Workfront-Berichten finden Sie [Erste Schritte mit Berichten](../../../reports-and-dashboards/reports/reporting/get-started-reports-workfront.md).

Beachten Sie beim Arbeiten mit Filtern in Listen oder Berichten Folgendes:

* Sie können vorhandene Filter in Listen und Berichten anpassen. Alle Benutzer, die über Berechtigungen für die Filter verfügen, können Ihre Änderungen auch sehen.

* Ihr Workfront-Administrator muss Ihnen Bearbeitungszugriff auf Filter, Ansichten und Gruppierungen gewähren, um dauerhafte Filter zu erstellen.

  Weitere Informationen finden Sie unter [Zugriff auf Filter, Ansichten und Gruppierungen gewähren](../../../administration-and-setup/add-users/configure-and-grant-access/grant-access-fvg.md).

* Die Berechtigungsstufe eines Filters bestimmt, wie Filter gespeichert werden. Wenn Sie den Filter ursprünglich erstellt haben, können Sie die Änderungen speichern. Andernfalls werden Sie aufgefordert, eine neue Version des zu ändernden Filters zu speichern.

  >[!TIP]
  >
  >Beachten Sie, dass sich Änderungen an einem Filter, den Sie für andere freigegeben haben, auch auf diese auswirken, wenn Sie ihn ändern.

* Sie können einen Filter, der für Sie freigegeben wurde, nur anpassen, wenn Ihnen der Benutzer, der ihn freigegeben hat, Zugriff gewähren konnte. Informationen zur Freigabe eines Filters finden Sie unter [Freigeben eines Filters, einer Ansicht oder einer Gruppierung](../../../reports-and-dashboards/reports/reporting-elements/share-filter-view-grouping.md).

## Filteroperatoren {#filter-operators}

Workfront verfügt über zwei Filteroperatoren, die jede Filteranweisung verbinden:

* **AND**: Wenn Sie zwei Filteranweisungen durch den AND-Operator verbinden, geben Sie an, dass beide Filteranweisungen gleichzeitig erfüllt werden sollen.

  Standardmäßig werden die Anweisungen in einem Filter durch den AND-Operator verbunden.

* **OR**: Wenn Sie zwei Filteranweisungen durch den OR-Operator verbinden, geben Sie an, dass eine der beiden Anweisungen erfüllt sein soll.

  >[!TIP]
  >
  >Wenn Sie Ihre AND-Anweisungen in OR-Anweisungen ändern, sollte die Anzahl der Elemente in Ihrem Bericht steigen.

## Filter-Modifikatoren {#filter-modifiers}

Sie können Filtermodifikatoren verwenden, um anzugeben, mit welcher Art von Informationen Ihre Daten übereinstimmen sollen.

>[!INFO]
>
>**Beispiel:**
>>Sie können nach Projekten filtern, die zu 100 % abgeschlossen sind. In diesem Fall können Sie den Modifikator „gleich“ in der folgenden Filteranweisung verwenden:
>
>`Project: Percent Complete Equal(Case Sensitive) 100`

Weitere Informationen zu Filtermodifikatoren finden Sie unter [Filter- und Bedingungsmodifikatoren](../../../reports-and-dashboards/reports/reporting-elements/filter-condition-modifiers.md).

## Erstellen von Filtern in der Builder-Benutzeroberfläche

Sie können einen Filter mithilfe der Standard- oder Legacy-Builder-Oberfläche wie folgt erstellen:

* Neu
* Vorhandenen Filter ändern

Informationen zum Erstellen eines Filters mithilfe der standardmäßigen oder veralteten Builder-Oberfläche finden Sie unter [Erstellen oder Bearbeiten von Filtern in Adobe Workfront](../../../reports-and-dashboards/reports/reporting-elements/create-filters.md).

## Erstellen von Filtern mithilfe der Textmodus-Oberfläche

Sie können Filter mithilfe der Textmodus-Oberfläche erstellen. Es wird empfohlen, die meisten Filteranweisungen mit der standardmäßigen oder veralteten Builder-Oberfläche zu erstellen und den Code des Filters nur als letzten Schritt zu bearbeiten. Es wird nicht empfohlen, einen Filter nur über die Textmodusschnittstelle von Grund auf neu zu erstellen.

Weitere Informationen zum Erstellen eines Filters mithilfe der Textmodusschnittstelle finden Sie unter [Bearbeiten eines Filters mithilfe des Textmodus](../../../reports-and-dashboards/reports/text-mode/edit-text-mode-in-filter.md).

## Erstellen von Filtern für komplexe Felder

Sie können Filter für komplexe Felder erstellen, z. B. Felder, deren Werte Kommas enthalten, und Felder, deren Werte ein benutzerdefiniertes Mehrfachauswahl-Feld sind.

### Erstellen von Filtern für Felder, deren Werte Kommas enthalten {#create-filters-for-fields-whose-values-contain-commas}

Beim Erstellen eines Filters im Textmodus und Filtern nach Feldwerten, die Kommas enthalten, müssen Sie einen Schrägstrich (“/„) vor den Kommas hinzufügen, die die Werte trennen, um sicherzustellen, dass der Wert als eine Filteroption gelesen wird. Dies gilt nur für die folgenden Feldtypen:

* Dropdown-Listen
* Optionsfelder
* Kontrollkästchen

Sie haben beispielsweise ein Optionsfeld für ein Projekt mit dem Namen „Farbe“ und die folgenden Optionen:

* Rot
* Blau
* Rot, blau

Wenn Sie einen Filter erstellen möchten, der nur Projekte findet, bei denen die dritte Option ausgewählt ist, sollte die Filteranweisung in der Textmodus-Oberfläche wie folgt lauten:

`DE:check=red/, blue`

`DE:check_Mod=in`

Diese Syntax stellt sicher, dass die Werte als eine Option zusammen gelesen werden. Wenn Sie den Schrägstrich weglassen, liest Workfront das Komma als ODER, wobei nur die Projekte mit der ersten oder zweiten Option ausgewählt werden.

Diese Syntax gilt auch bei der Erstellung von Eingabeaufforderungen. Weitere Informationen zur Verwendung von Eingabeaufforderungen finden Sie unter [Hinzufügen einer Eingabeaufforderung zu einem Bericht](../../../reports-and-dashboards/reports/creating-and-managing-reports/add-prompt-report.md).

### Erstellen von Filtern, deren Werte ein benutzerdefiniertes Mehrfachauswahl-Feld sind {#create-filters-for-whose-values-are-a-multi-select-custom-field}

Sie können Berichte zu benutzerdefinierten Feldern mit mehreren Auswahlmöglichkeiten erstellen. Beispielsweise können Sie Berichte zu Feldern erstellen, die Kontrollkästchen sind.

Wenn Sie jedoch Ergebnisse ausschließen möchten, für die nur eine der Optionen ausgewählt ist, zeigt der Bericht alle Objekte an, für die diese Option und eine andere Option ausgewählt sind.

Wenn Sie beispielsweise ein Feld mit drei Optionen (A, B und C) haben und 2 der drei Optionen (A und B, aber nicht C) für ein Objekt auswählen, können Sie einen Bericht mit einem Filter erstellen, der für die Optionen A und B den Qualifizierer „Nicht gleich“ aufweist, und es werden nur Projekte herausgefiltert, die sowohl A als auch B, aber nicht nur A und nicht nur B ausgewählt haben. Wenn Sie einen Filter mit einem Qualifizierer für Ungleich für Option A erstellen, werden nur Objekte herausgefiltert, für die nur A ausgewählt ist. Wenn A und B ausgewählt sind, werden diese Objekte jedoch weiterhin im Bericht angezeigt.

## Einschränkungen beim Zusammenführen mehrerer Filterregeln

Beim Erstellen eines Filters in Workfront können Sie nur fünf Objekte referenzieren, das Objekt des Berichts ausschließen.
